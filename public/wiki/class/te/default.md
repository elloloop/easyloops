# Class

## Class ante enti?

**Class** ante objects create cheyadaniki blueprint or template. OOP lo class attributes (data) and methods (behavior) define chesthundi. Class ni cookie cutter laga, objects ni cookies laga alochinchandi.

## Key Concepts

### Blueprint

Class defines what data and operations untayi, kaani object create cheyadu.

### Encapsulation

Data and operations okka unit lo bundle chesthundi.

### Abstraction

Complex details hide chesi, necessary parts matrame expose.

### Reusability

Once define aithe, chaala objects create cheyochu same structure tho.

## Class Components

### Attributes (Properties)

Data hold chese variables.

```
Car class attributes:
- color
- brand
- year
- speed
```

### Methods

Objects perform chese operations.

```
Car class methods:
- start()
- accelerate()
- brake()
- stop()
```

### Constructor

New objects create chesnapudu call aina special method.

## Class vs Object

| Class        | Object       |
| ------------ | ------------ |
| Blueprint    | Instance     |
| Definition   | Actual thing |
| Created once | Many times   |
| Example: Dog | Buddy, Max   |

Example:

- **Class**: Cookie recipe
- **Object**: Actual cookies

## Enduku Use Cheyali?

1. **Organization**: Related data together
2. **Reusability**: Define once, use many times
3. **Modularity**: Separate concerns
4. **Maintainability**: Changes easy
5. **Real-World Modeling**: Real entities map easy

## Access Modifiers

- **Public**: Anywhere access
- **Private**: Only within class
- **Protected**: Class and subclasses

## Static vs Instance

- **Instance Members**: Each object ki unique
- **Static Members**: Class level, all share

## Best Practices

1. **Single Responsibility**: One purpose
2. **Meaningful Names**: Clear names (PascalCase)
3. **Encapsulation**: Private data
4. **Cohesion**: Related things together
5. **Loose Coupling**: Minimal dependencies
